一个初学者的自白。为什么要写博客，为什么选择Hexo，以及为什么选择GitHub Pages。

Hello, Hexo.

Starter’s start

作为一个初学者，起步是不安、谨慎又兴奋的。婴儿学步总是蹒跚，新司机上路总是握紧方向盘。

Why Blog

想写个人博客的念头也在脑海里盘旋一阵了：

  • 一是因为不少掉过的坑很有记录的需求，尽管一些小坑可以记在daily notes里，但是一些大且重要的坑需要系统的复盘总结，才会记忆得更深刻，以后遗忘了再找出来温习也更方便。
  • 二是因为写博客的过程中可以学习实践一个不小的技术栈，不管是初期搭建部署博客时用到的 Hexo 和 GitHub Pages 框架，还是搭建后撰写blogs期间用到的 Markdown 排版知识和网页前端 HTML/CSS/JavaScript 技术，都是平时做数据开发不太会接触到的，通过写博客的机会正好多一些接触，扩宽知识面。
  • 三是因为下班回家有些无聊，找点事做。

Why Hexo

选择 Hexo 的原因很简单：作为一个颜控，博客漂亮是必选项。
在进行了多方面的比较，最后在颜值先行的指导思想下，综合考虑轻量化程度、插件丰富度、可扩展性等方面，选择了 Hexo 作为自己博客的框架。

Why GitHub Pages

部署在 GitHub Pages 上主要是因为这很方便也很省钱，不用去买服务器，用 Hexo 几个指令部署到 GitHub 上就可以生成静态博客页面。当然，坏处就是国内上 GitHub 的网速并不稳定，有时候 git clone 的速度能降到可怜的几 k/s，然后再报个错误中断下载，比较考验心态。
本来想在阿里云买个域名作为博客地址，而且个人实名认证也已通过，域名注册又要独立的实名认证，提交了好几遍都是失败，让人很有挫败感。同时阿里云主页页面操作很不流畅，每次登陆控制台都是煎熬。结果转向使用 腾讯云 之后，不仅页面不卡顿，认证也非常的便捷快速，满分好评。

Learn to learn

学会如何学习，也是一门学问。
几年前学习 C#/C++/Java 这些编程语言的时候，其实也想学好，但是光看书以为自己懂语言特性了，实际做项目的时候就不知从何入手，从而怀疑自己适不适合，有没有能力。当时的困难在于没有给自己规划一个合理的学习路径，也没有找一个人来告诉自己这一点。
后来在接触到 Python 和 Data Science 之后，发现不知不觉就初窥门径了。一开始一定要从简单又好玩的开始入门，特别是 《Python for Data Analysis》 这本书，真正引起了我学习大数据、学习编程语言的热情。原来还有这么简单又强大的处理数据的方法，简直太有趣了，这是当时看完这本书的想法。
然后一方面往算法、数据挖掘、机器学习等方向深入学习，一方面往 Python/Java 这些编程方向深入学习，中间也会遇到很多的岔路口，比如量化分析、前端 HTML/CSS、网络爬虫、Hadoop 全家桶、Spring 全家桶等等，有的发现自己很感兴趣就在新方向深入学习下去，有些则浅尝辄止以后再说。
开始之后，其实该怎么提升都会变得清晰起来，剩下的就是选择若干方向，以兴趣作舟，畅游这片无边海洋了。
